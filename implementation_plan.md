# implementation_plan

## goal
- 새로고침 시 시간이 한꺼번에 줄어드는 문제의 원인을 제거한다.
- 카카오맵과의 시간 오차를 줄일 수 있는 구조로 수정한다.
- `60초 이하 -> 곧 도착`, `API 상태 문구가 있을 때만 도착 표시` 규칙을 유지한다.
- 구현 후 제로베이스 관점에서 다시 점검한다.

## scope
- Worker와 Android 서비스 사이의 시간 보정 책임을 한쪽으로 정리한다.
- 현재 `barvlDt`, `rawBarvlDt`, `recptnDt`, `updatedAt` 사용 방식을 다시 정리한다.
- 카카오맵과 차이가 날 수 있는 원인들을 분류하고, 현재 구조에서 줄일 수 있는 것만 반영한다.
- 즐겨찾기 전환 후 빈 패널 문제도 같이 유지 보수한다.

## work steps
- [x] 1. Worker의 시간 보정 로직과 Android 서비스의 시간 감소 로직을 다시 비교한다.
- [x] 2. 이중 보정 구조를 제거한다.
- [x] 3. 패널 시간 계산 기준을 한 군데로 통일한다.
- [x] 4. 카카오맵과 시간 차이가 생기는 원인을 구조적으로 정리한다.
- [x] 5. 현재 공개 API 범위에서 줄일 수 있는 오차만 반영한다.
- [ ] 6. `보라매`, `강남`, `이문` 기준으로 다시 점검한다.
- [x] 7. 제로베이스 체크리스트로 재검토한다.

## verification criteria
- 새로고침 후 시간이 수십 초 단위로 급감하는 현상이 줄어들어야 한다.
- 패널 시간 계산 기준이 Worker/Android 중 한쪽으로 명확히 정리돼 있어야 한다.
- `도착`은 API 상태 문구가 있을 때만 표시돼야 한다.
- 카카오맵과 완전히 같진 않아도, 현재보다 시간 오차가 줄어들어야 한다.

## completion condition
- 시간 급감 원인이 제거되고, 시간 계산 구조가 단순하고 설명 가능해야 한다.
- 남은 일은 실제 기기 재검증과 미세 조정만 남아 있어야 한다.
