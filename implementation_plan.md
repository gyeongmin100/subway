# implementation plan

## goal
- 즐겨찾기별로 도착 스냅샷과 카운트다운 기준 시각을 따로 유지한다.
- 다른 즐겨찾기로 이동했다가 돌아와도 시간이 초기화되거나 다시 올라가 보이지 않게 만든다.
- 현재 규칙인 `도착순 2개`, `5초 자동 갱신`, `arvlCd 0/1 우선 표시`는 유지한다.

## scope
- `app/android/app/src/main/java/com/gyeongmin100/subway/SubwayPanelService.kt`

## work steps
- [x] 1. 현재 `currentArrivals` 단일 상태를 `favoriteId별 arrival cache` 구조로 바꾼다.
- [x] 2. 즐겨찾기 이동 시 기존 즐겨찾기 스냅샷을 지우지 않도록 바꾼다.
- [x] 3. 현재 즐겨찾기의 표시 목록은 캐시에서 읽고, 카운트다운은 그 캐시 기준 시각으로 계속 흐르게 한다.
- [x] 4. 새 API 응답이 오면 현재 즐겨찾기 캐시만 갱신한다.
- [x] 5. 현재 즐겨찾기 기준 가장 먼저 오는 2개만 계속 표시한다.
- [ ] 6. 전환 후 복귀 시 시간 이어짐과 새로고침 동작이 깨지지 않는지 점검한다.

## verification criteria
- A역에서 시간이 내려가다가 B역으로 갔다가 다시 A역으로 와도 시간이 다시 올라가지 않는다.
- 각 즐겨찾기의 카운트다운이 독립적으로 유지된다.
- 자동 갱신 5초 주기와 `arvlCd 0/1` 우선 표시 규칙이 유지된다.

## completion condition
- 즐겨찾기 전환 후 복귀 시 시간이 초기화되지 않는다.
- 패널이 즐겨찾기별로 살아 있는 스냅샷을 유지한다.
