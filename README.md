# Subway

앱을 열 시간도 아까운 사용자를 위해 만든 지하철 도착정보 앱입니다.  
핵심은 `앱 진입`이 아니라 `즉시 확인`입니다. 자주 이용하는 역을 즐겨찾기로 등록해두면, Android에서는 상단 알림 패널에서 바로 도착정보를 확인할 수 있습니다.

## 프로젝트 한 줄 소개

`Subway`는 서울 지하철 실시간 도착정보를 검색하고, 자주 보는 역을 즐겨찾기로 저장한 뒤, Android 알림 패널에서 빠르게 확인할 수 있도록 설계한 모바일 앱입니다.

## 왜 만들었는가

기존 지하철 도착정보 확인 흐름은 보통 아래처럼 깁니다.

1. 휴대폰 잠금 해제
2. 앱 찾기
3. 앱 실행
4. 역 검색
5. 도착정보 확인

출근길이나 환승 중에는 이 몇 단계조차 번거롭습니다.  
이 프로젝트는 이 문제를 `앱을 열지 않고도 알림 패널에서 바로 확인할 수 있게 만드는 것`으로 해결하려고 했습니다.

비유하면, 필요한 정보를 찾으러 서랍을 여는 앱이 아니라 책상 위에 바로 올려두는 앱에 가깝습니다.

## 핵심 가치

- 앱을 열지 않아도 빠르게 확인할 수 있어야 한다.
- 자주 보는 역은 다시 검색하지 않아야 한다.
- 화면이 복잡하지 않고, 필요한 정보만 짧게 보여야 한다.
- 실시간 데이터는 서울시 지하철 도착정보 API 기준으로 직접 받아와야 한다.

## 주요 기능

### 1. 역 검색
- 역 이름으로 지하철역을 검색할 수 있습니다.
- 노선과 방향을 함께 선택하는 구조라, 같은 역이라도 원하는 방향 기준으로 즐겨찾기를 나눌 수 있습니다.

### 2. 즐겨찾기 저장
- 자주 이용하는 역과 방향을 즐겨찾기로 저장할 수 있습니다.
- 저장된 항목은 다시 검색하지 않고 바로 접근할 수 있습니다.
- 현재 대표 즐겨찾기를 바꾸거나, 순서를 조정하거나, 삭제할 수 있습니다.

### 3. 실시간 도착정보 조회
- 서울 열린데이터광장의 지하철 실시간 도착정보를 기반으로 데이터를 조회합니다.
- 노선과 방향 기준으로 필요한 열차만 필터링해서 보여줍니다.
- 남은 시간은 초 단위 값을 기반으로 화면에서 카운트다운 형태로 보정합니다.

### 4. Android 알림 패널 연동
- 이 프로젝트의 메인 기능입니다.
- 즐겨찾기 상태를 Android 네이티브 패널 모듈과 동기화합니다.
- 사용자는 앱을 다시 열지 않아도 상단 알림 패널에서 주요 도착정보를 바로 확인할 수 있습니다.
- 바쁜 상황에서 `앱 실행`이라는 한 단계를 줄이는 것이 이 앱의 가장 큰 사용자 가치입니다.

### 5. 로컬 저장
- 즐겨찾기 목록과 현재 선택된 대표 역을 로컬에 저장합니다.
- 앱을 다시 켜도 이전 상태를 이어서 사용할 수 있습니다.

## 동작 구조

전체 흐름은 아래와 같습니다.

1. 사용자가 앱에서 역을 검색한다.
2. 원하는 노선/방향을 즐겨찾기로 저장한다.
3. 앱은 즐겨찾기 상태를 로컬 저장소와 Android 네이티브 패널에 동기화한다.
4. 패널 또는 앱에서 도착정보가 필요할 때 Worker API를 호출한다.
5. Worker는 서울시 Open API에서 데이터를 받아 정리한 뒤 앱이 쓰기 쉬운 형태로 반환한다.

즉, 모바일 앱은 사용자 경험과 상태 관리에 집중하고, Worker는 외부 API 정리와 응답 표준화 역할을 맡습니다.

## 저장소 구조

```text
.
├─ app/                # Expo / React Native 앱
│  ├─ android/         # Android 네이티브 패널 서비스 코드
│  ├─ assets/          # 앱 아이콘, 스플래시 이미지
│  └─ src/
│     ├─ components/   # 화면 컴포넌트
│     ├─ data/         # 역 검색용 정적 데이터
│     ├─ lib/          # 검색, 저장, API, 패널 연동 로직
│     └─ types/        # 타입 정의
├─ shared/             # 앱과 Worker가 공유하는 규칙
├─ workers/            # Cloudflare Workers 백엔드
│  └─ src/
└─ README.md
```

## 핵심 파일 설명

- `app/App.tsx`
  앱의 전체 상태와 화면 전환을 관리합니다.

- `app/src/lib/nativePanel.ts`
  Android 네이티브 패널 모듈과 통신하는 핵심 파일입니다. Android 전용 기능은 여기서 플랫폼 분기 처리합니다.

- `app/src/lib/search.ts`
  역 검색 인덱스와 검색 로직을 담당합니다.

- `app/src/lib/storage.ts`
  즐겨찾기와 현재 선택 상태를 AsyncStorage에 저장합니다.

- `workers/src/arrivals.ts`
  서울시 도착정보 API를 호출하고, 앱이 쓰기 쉬운 응답 형식으로 정리합니다.

- `shared/subwayRules.ts`
  노선명 정규화, 방향 처리 등 앱과 Worker가 함께 쓰는 규칙을 정의합니다.

## 실행 방법

### 1. 앱 실행

```bash
cd app
npm install
npm start
```

Android 개발 실행:

```bash
cd app
npm run android
```

iOS 실행 스크립트도 포함되어 있습니다:

```bash
cd app
npm run ios
```

다만 이 프로젝트의 대표 기능인 `알림 패널 바로 확인` 경험은 Android 중심으로 구현되어 있습니다.

### 2. Worker 실행

```bash
cd workers
npm install
npm run dev
```

배포 전에는 Cloudflare Worker 시크릿으로 `SEOUL_SUBWAY_API_KEY`를 설정해야 합니다.

```http
GET /api/arrivals?station=강남
```

응답에는 역 이름, 갱신 시각, 열차 목록, 노선 정보 등이 포함됩니다.

## Android와 iOS 차이

이 저장소는 하나의 코드베이스를 기준으로 유지합니다.  
검색, 즐겨찾기, API 호출, 저장 로직은 대부분 공용입니다.

다만 아래는 플랫폼 차이가 있습니다.

- Android
  - 알림 패널 기반 확인 경험 지원
  - 네이티브 패널 서비스 연동

- iOS
  - 앱 자체는 대응 가능
  - Android 알림 패널과 같은 방식의 경험은 동일하게 제공되지 않음
  - 필요 시 위젯, Live Activity, 알림 중심 UX로 별도 설계 필요

즉, 앱을 두 개 따로 만드는 구조가 아니라, 공통 코드는 같이 쓰고 플랫폼별 차이만 필요한 부분에서 분기하는 구조입니다.

## 이 프로젝트에서 중요하게 본 점

- 사용자가 앱을 열지 않아도 되게 만드는 것
- 자주 보는 역을 반복 검색하지 않게 만드는 것
- 실시간 정보를 짧고 빠르게 읽히게 만드는 것
- 데이터 가공 로직을 앱과 백엔드에서 역할 분리하는 것

기능 자체보다도, `행동 수를 줄여서 더 빠르게 확인하게 하는 경험`이 이 프로젝트의 중심입니다.
